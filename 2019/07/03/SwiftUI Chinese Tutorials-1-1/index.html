今年的 WWDC 上，苹果带来了重磅更新——SwiftUI，作为一名iOS开发，已经被 Apple 的 UI 开发方式折磨了很久，SwiftUI 对我们来说是一个好消息（更希望 SwiftUI 能把手伸到 Android 的口袋里）。
我根据苹果的 SwiftUI Tutorials 自己翻译整理这个教程，能力一般水平有限，可能翻译得不是很到位，但希望能够帮助到入门的开发者。

前言

由于我也是一边学习 SwiftUI 一边翻译，我会尽量在“说人话”和“严谨贴合原文”之间找到一个平衡，时不时也会穿插我自己的看法和理解。教程中有翻译不对或者我理解有偏差的地方还请见谅，谢谢~

下面我们正式开始

SwiftUI 要领 - 第一章：创建并组合 View

Creating and Combining Views

本教程将会通过引导你构建一个叫做 Landmarks 的应用来熟悉 SwiftUI 。Landmakrs 是一个可以发现和分享你喜欢的地点的 iOS 应用。你将通过创建展示地标详情的视图开始这段教程。

为了布局这些视图，Landmarks 使用了stacks 来把图片视图组件和文案视图组件组合并起来并分层。要添加一个地图到视图中时，你需要把一个标准的 MapKit 组建包含到工程中。为了让你精简视图设计，Xcode提供了实时反馈，所以你可以看到它们的变化是如何转换到代码中去的。

下载项目文件 来开始创建这个工程，并按照下面的步骤开始动手吧。

第一节：创建新工程并探索画布

创建一个新的Xcode工程并使用SwiftUI（在创建工程时勾选上SwiftUI选项，这里我就不放视频了）。探索一下画布、预览和SwiftUI实例代码。

要在Xcode上预览画布中的视图并与其交互，需要保证你的Mac运行了macOS 10.15 beta 版本的系统。

第1步


打开Xcode，在启动界面中点击 Create a new Xcode project，或者在顶部的工具栏中选择 File > New > Project。

第2步


在模板选择器中，选择 iOS 作为开发的平台，选择 Single View App 模板，然后点击 Next。

第3步


输入”Landmarks”作为新工程的名称，把 Use SwiftUI 复选框的勾勾打上，然后点击 Next 。在你的 Mac 上选择一个路径存放这个 Landmarks 工程。

第4步

在项目工程导航栏中，点击选择 ContentView.swift 这个文件。

默认情况下，SwiftUI View 文件会声明两个结构体。第一个结构体遵循 View 协议，并且描述了视图的内容和布局。第二个结构体为这个视图声明了预览。

第5步

在画布中，点击 Resume 来显示预览。

小提示
如果画布没有显示出来，在顶部工具栏中选择 Editor > Editor and Canvas 就能显示了。

第6步

在 body 属性中，把 “Hello World” 修改为对一句你自己问候。

当你在一个视图的 body 属性中修改代码时，预览会更新，把你的修改反映出来。

第二节：自定义 Text View

你可以通过修改你的代码来自定义一个视图的显示，或者使用 “inspector”（检查器） 来看看那些能够帮助你写代码功能，它们也可以用来自定义视图的显示。

当你创建 Landmarks 应用时，你可以任意将下列编辑器组合起来使用：源码编辑、画布或者检查器。不论你选择了哪种工具，你的代码一直都会保持更新。

接下来，你将会使用检查器来自定义 text view。

第1步

在预览中，Command键+鼠标点击你之前写好的问候语来弹出结构化的编辑弹框，并且选择 inspect。

这个弹框显示了一些不同的属性来让你编辑，这些属性取决于你所检查（inspect）的 view 的类型

第2步

使用检查器来修改 “Turtle Rock” 的文案，这将是展示在你应用中的第一个地标名称。

第3步

把字体修饰器的值改为 Title。

这将会把系统字体应用到 text 视图上，所以 text 能正确地对用户首选的字体和相关设置响应。

第4步

手动编辑代码来添加 color(.green) 修饰器；这将会把 text 视图的字体颜色变为绿色。

在自定义一个 SwiftUI View 控件时，你调用的那些方法被称为 modifiers（修饰器）。修饰器是包装了修改一个 view 显示或其属性的方法。每一个修饰器方法都返回了一个新的 view，所以多个修饰器可以共同用链式的方式串起来，纵向排列。

你的代码一直都是 view 真正的来源，当你使用检查器修改或移除一个修饰器时，Xcode 会立刻更新你的代码来适配当前的修改。

第5步

现在，在代码编辑器中通过 Command + 鼠标点击 Text 的声明来打开检查器，并且在弹框中选择 Inspect。点击 Color 弹出菜单并选择 Inherited 来把 text 字体颜色修改回黑色。

第6步

要注意 Xcode 会自动更新你的代码来反映控件变化，现在 color(.green) 修饰器已经被移除了。

第三节：使用 stack 来把 view 组合起来

除了你在前一节教程中创建的 view 之外，你还将添加另一些 text view 来显示地标的详情，例如公园的名称和它所在的州。

当创建一个 SwiftUI view 时，你只需要描述它所包含的内容、布局以及它在该 view 的 body 属性中的行为；但是 body 属性只会返回单个 view。当多个 view 是水平成组、垂直成组或前后重叠成组的时候，你可以把多个 view 通过 stack 组合和嵌入在一起。

在本节教程中，你将会使用一个垂直的 stack 来把标题放置在一个水平的 stack 上方，这个水平的 stack 包含了公园的详情。

你可以使用 Xcode 的结构化编辑支持功能将 view 嵌入到 view 容器当中，也可以打开一个检查器，或者来帮助你有效地完成其他修改。

第1步

用Command + 鼠标点击 text view 的初始化器，就能让结构化编辑弹窗显示出来，然后选择 Embed in VStack。

紧接着，你将通过从视图控件库中拖拽一个 Text 视图的方式，添加一个 text view 到 stack 中去。

第2步

在 Xcode 窗口的右上角点击加号按钮 (+) 打开视图控件库，然后拖一个 Text 视图到你代码里，就放在 “Turtle Rock” text view的后面。

第3步

用 Joshua Tree National Park 替换掉 Text 视图的占位文案。

接着，通过自定义公园的位置来匹配预想的布局

第4步

把公园的位置设置成 subheadling （副标题）。

第5步

编辑 VStack 初始化器来将它包含的视图控件左对齐。

默认地，stack 会把它的内容沿着轴线居中，并且提供适应上下文的合理边距。

接下来，你将要把另一个 text view 添加到公园位置文案的右边，用来显示公园所在的州。

第6步

在画布中，用Command + 鼠标点击 Joshua Tree National Park，并选择 Embed in HStack。

第7步

在位置文案后面添加一个新的 text view，把它的占位文案替换为公园所在的州的名称，并且将这个 text view 的字体设置为 subheadline。

第8步

为了让视图的布局恰好可以充满设备屏幕的宽度，要在包含有水平的 stack 里添加 Spacer 来将公园和州名这两个 text view 分隔开。

一个 spacer 会让它包含的 view 撑满父控件的所有空间，而不是仅仅根据它包含的内容来定义其大小。

第9步

最后，使用 padding() 修饰器方法来给地标的名称、详情一点小小的空间，让它们不那么紧贴屏幕边缘。

第四节：创建一个自定义的 Image View


地标名称和位置视图控件都已经设置好了，接下来要做的就是为地标添加一张图片。

并不需要在文件中添加更多的代码，你只需要创建一个自定义的 view，并把 mask、border 和 shadow 应用在图片上就可以了。

让我们从添加一张图片到工程的 asset 目录下开始。

第1步

从你下载的工程文件目录下的 Resource 文件夹中找到名为 turtlerock.png 的图片文件，把它拖放到 asset 目录的编辑器中去。Xcode 会为这张图片创建一个新的 image set（图集）。

现在，你要为自定义的 image view 创建一个新的 SwiftUI View。

第2步

在顶部工具栏中选择 File > New > File 来打开模板选择器。在 User Interface 分组中，点击选择 SwiftUI View ，然后点击 Next。把新的 SwiftUI View 文件命名为 CircleImage.swift 并点击 Create 创建文件。

此时你已经准备好了插入图片并修改它的显示方式来匹配之前预想好的设计。

第3步

使用 Image(_:) 初始化器来初始化 Turtle Rock 的图片，把占位的text view 替换掉。

第4步

调用 ClipShape(Circle()) 修饰器来把图片裁剪成圆形。

在你使用 mask （遮罩）时，Circle 可以作为一种形状的类型，或者在你在使用 stroke （描边）或 fill （填充）时，Circle 可以形成一个圆形的 view。

第5步

再创建一个灰色描边的圆形，然后把它用 overlay 覆盖到之前添加的图片上，来给图片形成一个边框的效果。

第6步

接下来，为图片添加一个半径为 10pt 的阴影。

第7步

把边框的颜色改为白色

到这里就完成 image view 的自定义效果了。

第五节：把 UIKit 和 SwiftUI View 结合起来

现在你已经做好创建一个 map view 的准备了。你可以使用 MapKit 中的 MKMapView 类来渲染一个地图控件。

要在 SwiftUI 中使用 UIView 的子类时，你要把其它的 view 包装成一个遵循 UIViewRepresentable 协议的 SwiftUI View。SwiftUI 还包含了针对 WatchKit 和 AppKit 相关的协议。

开始前，你要新创建一个能够呈现 MKMapView 的自定义 view。

第1步

在顶部工具栏中选择 File > New > File ，选择 iOS 平台，选择 SwiftUI View 模板，然后点击 Next。把新文件命名为 MapView.swift 并点击 Create 创建文件。

第2步

导入 MapKit，并且声明 MapKit 类型遵循 UIViewRepresentable 协议。

暂时不用管 Xcode 显示的报错，接下来几步你就能消除这些 error。

为了遵循 UIViewRepresentable 协议，你要添加两个必要的方法：UIView(context:) 方法和 updateUIView(_: context:) 方法。UIView(context:) 方法用来创建 MKMapView；updateUIView(_: context:) 方法用来配置 view 和响应变化。

第3步

把 body 属性用 UIView(context:) 方法替换掉，在 UIView(context:) 方法中创建并返回一个空的 MKMapView。

第4步

创建 updateUIView(_: context:) 方法，利用正确的坐标来把 Turtle Rock 设置到地图的中心，并确定地图合适的 region （地图显示的区域大小）。

当预览为 static（静态模式）时，只会渲染 SwiftUI View。因为 MKMapView 是一个 UIView 的子类，你要把预览切换成 live（实时模式）才能看到地图。

第5步

点击 Live Preview 按钮来把预览切换成实时模式。有可能也要在预览上方点一下 Try Again 或 Resume按钮才能显示实时的预览。

你会立刻看到 Joshua Tree National Park 的地图，Turtle Rock 就位于这个公园内。

第六节：把详情 view 组合起来

目前你已经有了所需的全部组件：地点的名称、圆形的图片和显示位置的地图。


结合目前使用的所有工具，你可以把这些自定义 view 组合起来创建一个地标详情视图，来完成我们最终的设计。

第1步

在工程的导航栏中，选择 ContentView.swift 文件

第2步

在之前创建好的 VStack 里面嵌入一个 VStack，让它把三个 text view 包起来。

第3步

把你自定义好的 MapView 添加到 stack 的最顶层。利用 frame(wdith:height:) 方法 MapView 的尺寸设置好。

当你只指定 height 参数时，view 会自动根据其内容来计算出高度。在这段代码中， MapView 会撑满所有可用的空间（水平方向）。

第4步

点击 Live Preview 按钮，来查看渲染好的地图控件。

当预览以 Live Preview（实时模式）显示时，你可以继续编辑这些 view。

第5步

把 CircleImage 视图添加到 stack 中去。

第6步

为了把 image view 布局到 map view 的上面，要在竖直方向上给 image view 设置 -130 pt 的偏移量，并且在 bottom（底部）留出 -130 pt 的 padding（边距）来跟其他 view 隔开。

把图片往上移动调整是为了给 text 上方留出一些空间。

第7步

在最外层 VStack 的底部添加一个 spacer ，把整体内容推到屏幕顶部。

第8步

最后，为了让 map 的内容可以伸展到紧贴屏幕上边缘，要在 map view 中添加一个 IgnoringSafeArea(.top) 的修饰器方法来忽略调 Safe Area 的上边距。

第一章到此结束，感谢你的阅读。
Have a nice day !!
